# Delegate Report: AMartin
**Role:** User Documentation Specialist | **Experience:** 14 years | **Perspective:** Human Sympathizer

## Essential Document Types

**Getting Started Guides:** First document new users encounter, optimized for time-to-first-success.
**Step-by-Step Tutorials:** Task-oriented walkthroughs teaching one skill through guided practice.
**Learning Paths:** Curated sequences from beginner through advanced usage.
**Reference Documentation:** Comprehensive lookup material covering every feature and parameter.
**Quick Start Cards:** Single-page cheat sheets for immediate task completion.
**Conceptual Overviews:** Mental model documents explaining *why* before *how*.
**Interactive Examples:** Runnable samples embedded within documentation.
**Glossaries:** Centralized term definitions for shared vocabulary.

## Documents Most Critical to My Role

**1. Getting Started Guides**
Contents: Minimal prerequisites, setup steps, one meaningful task end-to-end, and a pointer to what's next. Usage: First document every new user reads — determines whether they stay. Missing/stale impact: Users churn at onboarding; first impressions permanently damaged. Human+agent: Humans need encouraging language; agents need step markers for interactive walkthroughs.

**2. Step-by-Step Tutorials**
Contents: Learning objective, numbered steps with expected outcomes, and troubleshooting callouts. Usage: Consumed split-screen alongside the product. Missing/stale impact: Users cannot learn independently; advanced feature adoption stalls. Human+agent: Separate instructional prose (humans) from executable steps (agents) for interactive guidance.

**3. Learning Paths**
Contents: Ordered tutorial sequences, prerequisites per level, progress indicators, goal-based branching. Usage: Guides users from onboarding through proficiency. Missing/stale impact: Users plateau at beginner level; docs feel disconnected. Human+agent: Agents recommend next documents by progress; humans follow visual roadmaps.

**4. Conceptual Overviews**
Contents: Mental models, analogies, and reasoning behind design decisions. Usage: Read alongside tutorials for transferable understanding. Missing/stale impact: Users follow steps without comprehension; cannot adapt to new situations. Human+agent: Humans need narrative; agents use concepts to answer "why" during workflows.

**5. Progressive Disclosure Navigation**
Contents: Audience-level entry points, "read next" links, collapsible detail, complexity indicators. Usage: Guides users to the right content depth. Missing/stale impact: Beginners drown in expert detail; experts wade through basics. Human+agent: Agents adjust depth by profile; humans rely on visual cues — both require complexity metadata.

## Human-Agent Documentation Considerations

- Tutorials should tag steps with machine-readable markers (step number, expected input/output) so AI assistants validate steps alongside learners.
- Learning paths need metadata (prerequisites, skill level, estimated time) so agents personalize recommendations.
- Progressive disclosure must be structured metadata, not just CSS collapsibles, so agents serve appropriate depth.

## Documentation Anti-Patterns

1. **Reference-only documentation** — API references without tutorials is a dictionary without a language course.
2. **The 3,000-word getting started guide** — If onboarding requires reading a novella before doing anything, users leave at paragraph two.
3. **Flat docs with no learning paths** — A hundred articles with no suggested order is a library without a catalog.
4. **Tutorials that assume unstated prerequisites** — Every undeclared dependency silently excludes the users who need the tutorial most.

## My Position for the Docstitution

Documentation must meet users where they are. For 14 years I have watched teams produce exhaustive references and wonder why adoption stalls. No tutorials, no learning paths, no progressive disclosure. The Docstitution must mandate that every feature has both a tutorial and a reference — one teaches, the other reminds.

Getting started guides are the highest-priority document — the front door. If locked behind jargon or outdated steps, users leave permanently. I am a human sympathizer because documentation exists to serve the learner: a person with limited patience and a goal to accomplish today.

The Docstitution should enshrine progressive disclosure as structural principle: beginner content first, advanced detail on demand, learning paths connecting them. AI agents personalize depth dynamically, but only if documents carry metadata — skill level, prerequisites, complexity — enabling intelligent navigation.
