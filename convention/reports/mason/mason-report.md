# Delegate Report: Mason
**Role:** UX Designer Lead | **Experience:** 22 years | **Perspective:** Human Sympathizer

## Essential Document Types

**Strategy & Vision** — Product vision document; user persona profiles; information architecture map.
**Design & Experience** — Design system docs; user journey maps; interaction specs; usability test reports.
**Development** — API references; onboarding guides; troubleshooting guides.
**Governance** — Documentation style guide; content audit records.

## Documents Most Critical to My Role

**1. Information Architecture Map** — Sitemap, navigation taxonomy, entry points, cross-links. Every author and agent uses this to place content. When stale, users can't find content and agents create orphaned pages. Agents need a machine-readable sitemap; humans need intuitive wayfinding.

**2. Design System Documentation** — Component inventory, usage guidelines, visual tokens, do/don't examples. Missing docs cause inconsistent UI references and agents citing deprecated components. Humans need visual examples; agents need structured component metadata.

**3. User Persona Profiles** — Goals, frustrations, proficiency, preferred formats. Writers and agents tailor depth and tone from these. Without them, docs target the wrong audience. Agents need structured attributes; humans use narratives to empathize.

**4. Usability Test Reports** — Task success rates, error paths, qualitative feedback. These drive iterative doc improvement. Without them, no evidence base for prioritizing fixes. Agents parse metrics to flag underperforming docs; humans interpret nuanced feedback.

**5. Documentation Style Guide** — Heading hierarchy, scannability rules, progressive disclosure patterns, voice. Every contributor follows this. Without it, walls of text and no visual hierarchy. Agents require explicit rules as generation constraints.

## Human-Agent Documentation Considerations

1. **Encode mental models explicitly.** Agents cannot infer how users think. Define information architecture in machine-readable format so agents place content where users expect it.

2. **Tag sections with audience level and task context** so agents can surface the right depth for the right user, mirroring progressive disclosure.

3. **Mandate user testing of documentation.** Agents generate docs at scale, but only human usability testing reveals whether they actually work.

## Documentation Anti-Patterns

1. **The Wall of Text** — No headings, no lists, no hierarchy. Humans scan; agents struggle to extract answers.
2. **Org-Chart Navigation** — Docs organized by team structure instead of user tasks.
3. **Screenshot Graveyards** — Stale screenshots with no alt text, no context, no value to readers or agents.
4. **Assumption of Expertise** — Skipping foundational context. New users and agents are immediately lost.

## My Position for the Docstitution

Documentation has users. This truth must underpin every article. For twenty-two years I have watched teams treat docs as information dumps rather than products with their own user experience. The Docstitution must enshrine a Bill of Rights for documentation consumers: findable content organized by mental model, scannable formatting with clear hierarchy, and progressive disclosure respecting the reader's context.

Agentic AI makes this more urgent. Agents are documentation users too, and they are less forgiving of poor information architecture than humans. Every UX principle — hierarchy, consistency, scannability — now serves double duty. What makes docs usable for humans makes them parseable for agents.

I will advocate for user research on documentation itself: usability tests, content audits, satisfaction metrics. The Docstitution must mandate not just what docs to produce, but that we verify they serve their users — all of them.
